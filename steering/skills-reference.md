# Skills Reference

Pi Dynamics includes 18 on-demand skills invocable via the `/` menu in Kiro chat.

## Skill Catalog

| Skill | Description |
|-------|-------------|
| `tdd-workflow` | Enforces test-driven development with 80%+ coverage (unit, integration, E2E). Use when writing new features or fixing bugs. |
| `coding-standards` | Universal coding standards for TypeScript, JavaScript, React, and Node.js. Use when starting projects or reviewing code. |
| `security-review` | Comprehensive security checklist and patterns. Use when adding auth, handling user input, or creating API endpoints. |
| `verification-loop` | Runs build, type check, lint, tests, security scan, and diff review. Use after completing features or before PRs. |
| `api-design` | RESTful API design patterns and best practices. Use when designing or refactoring APIs. |
| `frontend-patterns` | React, Next.js, and frontend architecture patterns. Use when building UI or optimizing frontend. |
| `backend-patterns` | Node.js, Express, and backend architecture patterns. Use when building APIs or backend infrastructure. |
| `e2e-testing` | End-to-end testing with Playwright or Cypress. Use when adding E2E tests. |
| `golang-patterns` | Go idioms, concurrency patterns, and best practices. Use when writing or reviewing Go code. |
| `golang-testing` | Go testing with table-driven tests and benchmarks. Use when writing Go tests. |
| `python-patterns` | Python idioms, type hints, and best practices. Use when writing or reviewing Python code. |
| `python-testing` | Python testing with pytest and coverage. Use when writing Python tests. |
| `database-migrations` | Database schema design and migration patterns. Use when creating migrations. |
| `postgres-patterns` | PostgreSQL-specific patterns and optimizations. Use when working with PostgreSQL. |
| `docker-patterns` | Docker and containerization best practices. Use when creating Dockerfiles. |
| `deployment-patterns` | Deployment strategies and CI/CD patterns. Use when setting up deployments. |
| `search-first` | Search-first development methodology. Use when exploring unfamiliar codebases. |
| `agentic-engineering` | Agentic software engineering patterns. Use when working with AI agents or building agentic systems. |

## Usage

1. Type `/` in Kiro chat to open the skills menu
2. Select a skill (e.g., `tdd-workflow` when starting a new feature)
3. The agent guides you through the workflow with specific instructions and checklists

## File Locations

After installation: `.kiro/skills/{skill-name}/SKILL.md`
